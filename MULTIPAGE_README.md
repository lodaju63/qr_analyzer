# 📱 QR 탐지 시스템 - 멀티페이지 버전

Streamlit 멀티페이지 구조로 구성된 QR 탐지 시스템입니다.

## 🎯 구조

```
qr_sh/
├── Home.py                    # 메인 홈 페이지
├── pages/
│   ├── 1__비디오_QR_탐지.py  # 비디오 QR 탐지
│   ├── 2__이미지_QR_QR_탐지.py  # 이미지 QR 탐지
│   └── 3__프레임_추출.py      # 프레임 추출
└── ...
```

## 🚀 실행 방법

### 단일 명령어로 실행
```bash
streamlit run Home.py
```

이제 **하나의 창**에서 모든 기능을 사용할 수 있습니다!

- 왼쪽 사이드바에서 페이지 선택
- 각 페이지는 독립적으로 동작
- 세션 상태는 페이지 간에 공유되지 않음 (독립적)

## 📑 페이지 구성

### 🏠 홈 페이지 (Home.py)
- 시스템 개요
- 각 도구에 대한 설명
- 빠른 시작 버튼

### 1️⃣ 비디오 QR 탐지
- 동영상 파일에서 QR 코드 탐지 및 해독
- 실시간 처리 및 추적
- 결과 비디오 저장

### 2️⃣ 이미지 QR 탐지
- 단일 이미지 처리
- 다양한 전처리 옵션
- 결과 이미지 다운로드

### 3️⃣ 프레임 추출
- 비디오 프레임 추출
- 프레임 탐색기
- 썸네일 그리드

## 🔄 기존 방식과의 비교

### ❌ 이전 방식 (별도 포트)
```bash
streamlit run web.py --server.port 8501      # 창 1
streamlit run img.py --server.port 8502      # 창 2
streamlit run videoframe.py --server.port 8503  # 창 3
```
- **3개의 창**, 3개의 프로세스
- 메모리 사용량 증가
- URL 관리 복잡

### ✅ 새로운 방식 (멀티페이지)
```bash
streamlit run Home.py
```
- **1개의 창**, 1개의 프로세스
- 사이드바로 페이지 전환
- 깔끔하고 체계적

## 💡 장점

1. **사용 편의성**: 하나의 창에서 모든 기능 접근
2. **효율성**: 메모리 사용량 감소
3. **확장성**: 새로운 페이지 추가 용이
4. **모듈화**: 각 페이지 독립 관리
5. **공식 지원**: Streamlit 공식 멀티페이지 방식

## 🔧 기존 파일과의 관계

원본 파일들은 그대로 유지됩니다:
- `web.py` - 원본 유지
- `img.py` - 원본 유지  
- `videoframe.py` - 원본 유지

`pages/` 폴더의 파일들은 멀티페이지용 복사본입니다.

## 📝 참고사항

- 각 페이지는 독립적으로 동작합니다
- 세션 상태는 페이지 간에 공유되지 않습니다
- 모델 로딩은 각 페이지에서 별도로 수행됩니다
- 파일명의 숫자(1_, 2_, 3_)가 사이드바 표시 순서를 결정합니다
